Using the list

The list page shows every ranked challenge, accepted records, verifier credits, level IDs, passwords, and verification videos.

  • Main list levels are ranks 1 through 75.
  • Extended list levels are ranks 76 through 150.
  • Legacy levels stay visible for history, but they do not accept new records after the grace period ends.

Use the search and rank filters on the list page to narrow results by level, creator, verifier, ID, or list section.

Submitting records

Main list records must meet the listed qualify percentage. Extended list records must be full completions.

  1. Do not use hacks. FPS bypass is allowed up to 360 FPS.
  2. Play the exact level ID shown on the list.
  3. Include source audio, clicks, or taps in the video.
  4. Show the previous attempt and the full death animation before the completion, unless the completion is on the first attempt.
  5. Show the player hitting the endwall.
  6. Do not use secret routes, bug routes, or easy modes.

When a level falls to Legacy, records are accepted for 24 hours after it falls off. After that, new records are closed.

Roulette

Roulette creates randomized practice runs from the available list data. It skips levels that do not have a playable verification video yet.

Roulette progress is saved locally in the browser, and runs can be imported or exported when you want to move them between devices.

Editing data

The public list order lives in data/_list.json. Each listed entry points to a level JSON file in data/ with the same name.

Level files

Use data/_example.json as the shape for new challenge entries. Listed levels should include ID, name, author, creators, verifier, verification, percent to qualify, password, and records.

Editor metadata

Editor credits live in data/_editors.json. Supported roles are owner, admin, helper, dev, and trial.

Validation

Run python3 scripts/validate_data.py before submitting data changes.
